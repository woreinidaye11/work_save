Transformations and uninformative priors

A conceptual difficulty with uninformative priors arises from transformations.
Suppose 0 < θ\thetaθ < 1 is a parameter for which we have no information.
We have previously stated it would be reasonable to assume θ∼U(0,1)\theta \sim U(0,1)θ∼U(0,1).
Now let ϕ=θ2\phi = \theta^2ϕ=θ2 so that 0 < ϕ\phiϕ < 1 and ,since we have no information about θ\thetaθ, we must also have no information about ϕ\phiϕ.
The previous "principle" would require ϕ∼U(0,1)\phi \sim U(0,1)ϕ∼U(0,1).
But this contradicts θ∼U(0,1)\theta \sim U(0,1)θ∼U(0,1).

Jeffreys' Invariance Principle

To resolve this difficulty, Jeffrets proposed the following principle for constructing uninformative priors.

Rejection sampling

Rejection sampling is a method for generating random observations from a density, f(θ)f(\theta)f(θ), that we cannot sample directly.
The main assumption is the existence of a known density, g(θ)g(\theta)g(θ) such that:

  1. it is easy to sample observations from the density g(θ)g(\theta)g(θ)
  2. you can explicity calculate a constant, K, such that Kf(θ)≤g(θ)Kf(\theta) \le g(\theta)Kf(θ)≤g(θ) for all θ\thetaθ

Formal algorithm

To sample form f(θ)f(\theta)f(θ)

  1. Draw a single observation , θ\thetaθ, from g.
  2. Accept (keep) a sample θ^\hat \thetaθ^ with probability Kf(θ)g(θ)\frac{Kf(\theta)}{g(\theta)}g(θ)Kf(θ)​ and reject (do not keep) with probability 1−Kf(θ)g(θ)1-\frac{Kf(\theta)}{g(\theta)}1−g(θ)Kf(θ)​
  3. Repeat steps 1 and 2 until an observation is accepted.

Key Issues for MCMC 1

Burn-in: since your initial condition for the Markov Chain is not "correct", i.e. not drawn from the stationary distribution ( which we don't know or we wouldn't be using the Gibbs sampler), your initial samples might no fit the distribution.
Burn-in: in theory no problem since the Markov Chain converges anyway. Best practice is trying to start your MCMC in a region of high probability. As secondary, discarding some initial steps from your Markov Chain can remove the so-called "burn-in" period where your Chain has disproportionately many samples in low probability regions.

Key Issues for MCMC 2

Correlation: Our samples are correlated for several steps. The issue is that if our samples stay correlated for a long time, then waht looks like many samples(100+), might actually be equivalent to many fewer uncorrelated samples.

ESS

Our samples are correlated for several steps. We'll need a diagnostic for the Effective Sample Size(ESS). The ESS estimates how many independent samples our MCMC sample is equivalent to: for S MCMC sample, 1≤ESS≤S1 \le ESS \le S1≤ESS≤S

Key Issues for MCMC 3

If the posterior distribution is multimodal with "islands" of high density surrounded by regions of very low density, transitions between the high density regions may require an unrealistically high number of steps.

for(i in 2:N){
  #sample from proposal
  proposal <- sample_proposal(samples[i-1,])
  
  #compute probability - possible to get Inf or NaN
  r <- (post_exact( proposal ) /
    post_exact( samples[i-1,] )) *
    (prop_dens( proposal, samples[i-1,] ) /
    prop_dens( samples[i-1,], proposal ))
  
  # Possible to get NaNs, e.g. if the proposal is a negative variance. 
  r[is.nan(r)]=0 #reject. 
  
  #accept or reject step
  prob <- runif(1,0,1)
  if( prob<r ){ #then accept
    samples[i,] = proposal
    n_accept <- n_accept+1
  } else {
    samples[i,] = samples[i-1,]
  } #end else
} #end for loop

Bayesian linear regression

Key is to translate the frequentist regression assumptions to a Bayesian form;
something like Y∼N(Xβ,σ2I)Y \sim N(X\beta,\sigma^2I)Y∼N(Xβ,σ2I)

Details

  1. must setup Bayesian model(prior,likelihood, posterior)
  2. Choose the likelihood Y∣X,β,σ2∼N(Xβ,σ2I)Y \mid X,\beta,\sigma^2 \sim N(X\beta,\sigma^2I)Y∣X,β,σ2∼N(Xβ,σ2I)
  3. Choose priors for β,σ( or σ2)\beta,\sigma (\textnormal{ or } \sigma^2)β,σ( or σ2)
  4. Compute β,σ2∣Y,X\beta, \sigma^2 \mid Y, Xβ,σ2∣Y,X. View X as an explanatory variable(fixed)

Posterior calculation

leave variable X condition on the right hand side then can get the bayes rule. In here we assume that knowing X does not help us knowing β,σ2\beta ,\sigma^2β,σ2, then we can convert to the second line

p(β,σ2∣Y,X)∝p(Y∣β,σ2,X)p(β,σ2∣X)p(\beta ,\sigma^2 \mid Y,X) \propto p(Y \mid \beta ,\sigma^2,X) p(\beta ,\sigma^2 \mid X) p(β,σ2∣Y,X)∝p(Y∣β,σ2,X)p(β,σ2∣X)

=p(Y∣β,σ2,X)p(β,σ2)= p(Y \mid \beta ,\sigma^2,X) p(\beta ,\sigma^2) =p(Y∣β,σ2,X)p(β,σ2)

Generally recover similar results to standard regression by choosing uninformative priors on β,σ2\beta ,\sigma^2β,σ2
